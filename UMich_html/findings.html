Findings

Tests run and conducted on the 4PROGRESS Cluster


The purpose of building the 4PROGRESS Cluster is to extend the compute capability of the desktop of one machine in the lab to code that can utilize all desktop machines in the 4PROGRESS lab. Utilizing multiple desktops also requires code that can accommodate and adapt for using multiple machines and different hardware landscapes. In particular, we attempted to optimize the GPU rendering portion of the deep scene estimation pipeline created by the graduate students in the 4PROGRESS lab.

The deep scene estimation pipeline is to build a semantic map at the level of scene graph for mobile manipulator.

Implementing OpenMPI and cluster computing, we were able to offload rendering onto multiple desktop machines. Unfortunately, this does come prevent any other user from using the desktop machines when utilizing the onboard Graphics card. If using only CPU required programs, all machines can be utilized by both remote hosts and the current local user.

Below are our recorded findings and how the cluster performed on several different levels.


Some areas to improve on is that message passing over a network does cost time when utilizing multiple GPUs on multiple machines rather than interposes communication between multiple GPUs on a single machine. Another area that also causes concern is the GPU native coding language, CUDA, does not allow for process communication between GPUs on separate machines, but does allow for communication between GPUs on a single machine. A rather, more cost effective way than utilizing a cluster would be to utilize a single machine with multiple GPUs. This would in turn cut down time spend on machine process communication, code restructuring overhead. However, this method does give new life to machines no longer being used, allowing more complex computations to be calculated within the lab rather than being outsourced by a third party.
